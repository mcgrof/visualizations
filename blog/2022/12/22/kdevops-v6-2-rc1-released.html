Dec 22, 2022 Linux / Kernel

kdevops v6.2-rc1 released

original source ↗

After three years since the initial announcement, version 6.2-rc1 of kdevops has been released. This project serves as a DevOps framework designed to streamline Linux kernel development workflows.

The framework has evolved significantly from its filesystem-testing origins. By adopting kconfig, kdevops now supports multiple workflows within a single codebase. Integration of jinja2 templating with Ansible and PCI-E passthrough support via kconfig has demonstrated the project's capacity for handling complex, emerging technologies.

New Technology Support

Two noteworthy technologies now have robust support: Zoned storage and CXL. The framework bundles necessary tooling to facilitate testing and development work.

CXL development workflows have been dramatically simplified. Setting up a complete Linux kernel development environment with meson unit tests and PCI-E passthrough now requires just six commands: 

make dynconfig
make
make bringup
make linux
make cxl
make cxl-test-meson

Development Activity

Over the past eight months, the project accumulated 680 commits from contributors including Adam Manzanares, Amir Goldstein, Chandan Babu R, Jeff Layton, Joel Granados, Josef Bacik, Luis Chamberlain, and Pankaj Raghav.

The project recently transitioned to the linux-kdevops GitHub organization to enable broader collaboration and scaling across different subsystems and workflows.
